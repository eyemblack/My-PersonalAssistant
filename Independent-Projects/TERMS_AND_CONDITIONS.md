# Terms & Conditions (เงื่อนไขการให้บริการ)

เอกสารแนบท้ายใบเสนอราคา เพื่อป้องกันความขัดแย้งและควบคุมขอบเขตงาน

## 1. การแบ่งชำระเงิน (Payment Terms)
- **มัดจำ (Deposit):** 50% ของมูลค่าโครงการ (ชำระก่อนเริ่มงาน)
- **งวดสุดท้าย (Final Payment):** 50% ของมูลค่าโครงการ (ชำระหลังตรวจรับงานและก่อนส่งมอบ Source Code)

## 2. ขอบเขตการแก้ไขงาน (Revision Policy)
- **จำนวนครั้ง:** สามารถแก้ไขรายละเอียดได้ **3 ครั้ง** ภายใต้ขอบเขตงานที่ตกลงกันไว้
- **ขอบเขต:** การแก้ไขหมายถึงการปรับเปลี่ยนหน้าตา (UI), แก้คำผิด หรือปรับ Logic เดิมที่วางไว้
- **การแก้ไขส่วนเกิน:** หากเกินจำนวนครั้งที่กำหนด คิดค่าบริการเพิ่มครั้งละ [ระบุราคา] บาท หรือตามตกลง

## 3. การเพิ่มฟีเจอร์นอกเหนือข้อตกลง (Scope Creep)
- หากมีการขอเพิ่มฟีเจอร์หรือหน้าเว็บไซต์ใหม่ที่ไม่ได้ระบุในใบเสนอราคา ผู้พัฒนาจะจัดทำใบเสนอราคาเพิ่มเติมแยกต่างหาก และอาจมีการปรับเปลี่ยนกำหนดการส่งมอบงานตามความเหมาะสม

## 4. ระยะเวลาการตรวจรับงาน (User Acceptance Testing)
- ลูกค้าต้องตรวจสอบและแจ้งแก้ไขงานภายใน **7 วันทำการ** หลังจากผู้พัฒนาส่งมอบงานแต่ละงวด หากไม่มีการแจ้งกลับภายในระยะเวลาดังกล่าว ให้ถือว่าลูกค้ายอมรับงานในงวดนั้นๆ โดยปริยาย

## 5. ทรัพย์สินทางปัญญา (Intellectual Property)
- เมื่อชำระเงินครบถ้วนแล้ว ลิขสิทธิ์ใน Source Code และทรัพย์สินต่างๆ จะตกเป็นของลูกค้า โดยผู้พัฒนามีสิทธิ์ในการนำภาพผลงานไปใช้เป็น Portfolio เท่านั้น
